# Publishing Notes

这一节记录的是：这个仓库在整理为 GitHub 参考仓库时，采用了哪些编辑和组织原则。

## 1. 仓库的整理重点

当前版本重点呈现 4 类内容：

- 论文写作工作流说明
- 分阶段计划文档
- 章节素材包结构
- 本地 skill 的来源、分工与改造方式

## 2. 编辑原则

整理这套仓库时，我优先遵循的是：

- 让工作流更容易被别人复用
- 让仓库内的文档关系更清晰
- 让 skill 来源和本地改造链路更明确
- 让示例结构优先于零散素材

## 3. 首版整理动作

`v1` 这一版主要做了这些收口：

- 清理了复制过程中的冗余缓存文件
- 把长 prompt 改写成 route、任务契约和本地化说明
- 统一了文档间的相对链接和导航路径
- 把阶段计划、skill 说明和材料结构整理成可连续阅读的仓库层次

## 4. 为什么要这样组织

这类仓库真正的价值，不在于堆很多 prompt 或很多技能名，而在于：

- 工作流足够连续
- 阶段切分足够清楚
- 章节材料组织可复用
- 不同 skill 的职责边界明确

## 5. 后续适合继续扩展的方向

- 增加更多流程复盘文档
- 增加更多 plan 模板和调用示例
- 增加更多“如何把 prompt 体系整理成本地 skill”的经验总结
- 增加更多图表、摘要、终稿打磨阶段的案例说明
