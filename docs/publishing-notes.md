# Publishing Notes

这一节记录把本仓库公开发布到 GitHub 时需要注意的边界。

## 1. 公开版保留什么

- 工作流说明
- 计划模板和阶段计划结构
- 章节素材包的层级设计
- 本地 skill 的路由设计与角色边界
- skill 来源与本地化改造思路

## 2. 公开版明确不保留什么

- 完整论文正文
- 原始实验数据
- 私有截图
- 第三方论文 PDF 原文
- 带本机绝对路径的私有工程记录
- 需要依赖我本地环境才能理解的未清理文档

## 3. 这次公开整理已经做了什么

- 删除了复制过程中的 `__pycache__`
- 删除了素材包里不适合公开的 PDF 原文
- 将私有绝对路径替换成占位路径或相对路径
- 将公开后失效的本机 Markdown 链接改成 GitHub 可用链接
- 将高风险的第三方长 prompt 原文改成公开安全版摘要

## 4. 公开版与私有工作版的关系

两者不是一回事。

私有工作版保留的是：

- 真实论文工程
- 真实素材全文
- 真实路径
- 真实实验归档

公开版保留的是：

- 方法论
- 流程结构
- 路由设计
- 公开安全的 skill 快照

## 5. 为什么公开版不追求“完全原样”

因为原样保留会带来至少三类风险：

- 本机路径与个人环境泄漏
- 第三方 prompt / skill 原文复用风险
- GitHub 页面上大量无效链接和不可公开资产

所以这次公开版的目标不是“原始工作副本”，而是“可安全分享的方法论仓库”。

## 6. 如果你想继续扩展这个仓库

优先方向：

- 增加更多公开安全的流程复盘文档
- 增加更多“如何把 prompt 体系压成 skill”的经验总结
- 增加更多示例性的计划模板

不建议直接加入：

- 私有论文正文
- 原始检测报告
- 未清理的实验资产
