# Publishing Notes

这一节记录把本仓库公开发布到 GitHub 时需要注意的边界。

## 1. 可以公开的内容

- `README`
- 工作流说明
- 计划模板
- 各章节素材包结构
- 本地自定义 skill
- skill 来源与改造说明

## 2. 不建议公开的内容

- 完整论文正文
- 未脱敏实验数据
- 私有截图
- 带隐私或版权风险的原始文件
- 不确认授权的第三方论文原文

## 3. 本仓库已经做的清理

为了更适合公开：

- 已删除复制过程中的 `__pycache__`
- 已删除素材包中复制过来的 PDF 原文
- 当前保留的重点是 markdown 结构化资产和本地 skill

## 4. 如果还要继续整理

建议再做两步：

1. 检查所有文档里是否还出现真实本地绝对路径
2. 把必要的示例路径改成相对路径或占位路径

## 5. 如果你想让它更像开源项目

建议再补：

- `LICENSE`
- `.gitignore`
- `CONTRIBUTING.md`
- 一份英文版 `README`

## 6. 一个更现实的公开策略

如果你不想一次性全部公开，可以分两层：

1. 公开版 GitHub 仓库
   - 只放工作流、计划模板、skill 说明、素材包结构

2. 私有版本地仓库
   - 保留论文正文、真实数据、完整实验归档、未清理路径

这样最稳。
