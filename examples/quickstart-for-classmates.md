# Quickstart For Classmates

如果你是第一次看这个仓库，最短上手路径如下。

## 1. 先理解一句话

这套方法不是“让 AI 一口气写完整篇论文”，而是“把论文写作拆成多个低风险阶段，然后让不同 skill 分工完成”。

## 2. 先看哪 3 个文件

先按顺序看：

1. [README.md](../README.md)
2. [docs/workflow-overview.md](../docs/workflow-overview.md)
3. [docs/skill-map.md](../docs/skill-map.md)

## 3. 如果你想复用我的方法

先照着准备这 3 类东西：

1. 长期 authority 文档
   - 大纲
   - 证据映射
   - 写作日志

2. 按章节的素材包
   - 可写层
   - 结果层
   - 核对层

3. 最小 skill 组合
   - `doc-coauthoring`
   - `graduation-thesis-editor`
   - `humanizer`

## 4. 如果你已经有初稿

不要直接全篇盲润。

推荐顺序是：

1. 用 `thesis-closeout-reviewer` 先找问题
2. 用 `graduation-thesis-editor` 修局部问题
3. 用 `academic-paper-reviewer` 做复审
4. 最后再用 `humanizer`

## 5. 如果你还没有开始写

不要上来就让 AI “写一篇论文”。

更稳的顺序是：

1. 先建 authority
2. 再做章节素材包
3. 再让 `doc-coauthoring` 起草各章
4. 再引入 `graduation-thesis-editor`

## 6. 最后记住一条

论文写得更稳，不是因为 prompt 更花，而是因为流程更清楚、边界更严格、修改更分层。
