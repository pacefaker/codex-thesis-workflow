# Quickstart

如果你是第一次看这个仓库，最短上手路径如下。

## 1. 先记住一句话

这套方法的重点不是让 AI 一次性写完整篇论文，而是把论文写作拆成多个可控阶段，再让不同 skill 在对应阶段各司其职。

## 2. 先看哪 3 个文件

建议先按顺序看：

1. [README.md](../README.md)
2. [docs/workflow-overview.md](../docs/workflow-overview.md)
3. [docs/skill-map.md](../docs/skill-map.md)

## 3. 如果你想搭自己的版本

先准备这 3 类基础材料：

1. 长期 authority 文档
   - 大纲
   - 证据映射
   - 写作日志

2. 章节素材包
   - 正文可写层
   - 结果分析层
   - 核对层

3. 最小 skill 组合
   - `doc-coauthoring`
   - `graduation-thesis-editor`
   - `humanizer`

## 4. 如果你已经有初稿

推荐顺序是：

1. 用 `thesis-closeout-reviewer` 先做问题诊断
2. 用 `graduation-thesis-editor` 修局部问题
3. 用 `academic-paper-reviewer` 做复审
4. 最后再用 `humanizer` 做轻量语言打磨

## 5. 如果你还在从 0 到 1 起草

推荐顺序是：

1. 先建 authority
2. 再做章节素材包
3. 再让 `doc-coauthoring` 起草各章
4. 再引入 `graduation-thesis-editor` 处理局部修补

## 6. 最后记住一条

论文写得更稳，往往不是因为 prompt 更花，而是因为流程更清楚、边界更严格、修改更分层。
