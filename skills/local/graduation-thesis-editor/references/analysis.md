# Public-Safe Analysis Routes

这一文件保留实验分析与 reviewer 式局部诊断的任务设计，不公开第三方长 prompt 原文。

## 实验分析

### 任务定位

- 输入：结果表、指标摘要、实验结论草稿、结果说明段
- 输出：更像论文正文的分析段落

### 公开版任务契约

- 所有结论必须严格基于输入结果
- 重点不是报数，而是比较、趋势、异常点和解释深度
- 如果没有明显优势，也要如实写
- 不允许编造“显著提升”或不存在的现象

### 典型输出风格

- 先给核心结论
- 再解释数值差异和趋势
- 最后回到该段在章节中的作用

## 论文整体以 Reviewer 视角进行审视

### 任务定位

- 输入：整节、整章或较大篇幅正文
- 输出：reviewer 风格的问题诊断与修文建议

### 公开版任务契约

- 区分致命问题和可修补问题
- 问题要具体到论证、实验、章节功能或表达缺陷
- 允许肯定真正有价值的贡献
- 输出应更接近“诊断报告”，而不是直接代写新正文

### 与 `thesis-closeout-reviewer` 的边界

- 如果是整篇论文收口，优先 `thesis-closeout-reviewer`
- 如果是局部大段、单章或 reviewer 视角局部压力测试，可以走这里

## 公开版说明

这一组 route 最值钱的地方不是某一段固定 prompt，而是：

- 它把“分析”和“表面润色”区分开了
- 它把“局部 reviewer 诊断”和“整篇 closeout”区分开了
