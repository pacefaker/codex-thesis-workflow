# Public-Safe Translation Routes

这一文件保留 `graduation-thesis-editor` 的翻译类 route 设计，但不公开第三方长 prompt 的逐字原文。

## 中转英

### 任务定位

- 输入：中文草稿、中文笔记、已有中文论文段落
- 输出：英文论文风格文本，必要时可保持 LaTeX 友好

### 公开版任务契约

- 保留原意、数字、术语、符号、公式
- 优先逻辑准确与学术表达清晰
- 若目标是 LaTeX，输出需保持纯净，不引入无关格式
- 需要时保留中文对照，便于核对语义是否漂移

### 本地化理由

原始来源里的价值不在“万能翻译”，而在于把“科研翻译 + 语义核对 + 论文语体约束”合并到了一个稳定任务里。

公开版只保留这类任务定义与输出契约，不保留原始长 prompt 正文。

## 英转中

### 任务定位

- 输入：英文论文段落、英文 LaTeX 片段
- 输出：用于理解和核对的中文解释文本

### 公开版任务契约

- 目标是帮助快速理解，而不是二次润色
- 去掉妨碍阅读的引用、标签等噪声时，要保持主句信息不丢失
- 数学与符号需要转成便于阅读的中文表述
- 输出以自然中文为主，不保留无意义的 LaTeX 噪声

### 本地化理由

这一路由的核心不是“翻得好看”，而是“让论文作者快速检查英文表述是否仍然忠于原意”。

## 中转中

### 任务定位

- 输入：中文要点、中文口语化草稿、碎片笔记
- 输出：更像论文正文的中文段落

### 公开版任务契约

- 优先做逻辑重组，而不是逐句表面润色
- 保持一个段落一个中心
- 避免把列表机械堆成模板句
- 不编造没有提供的事实、结论、引用

### 本地化理由

这个 route 在真实论文中非常高频，因为很多原始输入并不是“段落”，而是零散中文笔记。

## 公开版说明

如果你要复刻这个 skill：

1. 保留这 3 个 route 的任务边界。
2. 保留“输入输出契约先于文风修饰”的原则。
3. 用你自己确认可公开的 prompt 实现替换原文。
