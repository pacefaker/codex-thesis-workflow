# Materials Package Structure

公开版没有附带我的私有章节素材全文，但保留了“章节素材包如何设计”的结构说明。

## 典型素材包分层

一章素材包通常至少分成下面几层：

- 正文可写层
- 结果分析层
- 后台核对层

## 这样分层的目的

- 把真正可写进正文的事实和后台核验信息分开
- 避免 run-level 细节直接污染论文正文
- 在重写某章时快速定位 authority

## 一个代表性结构

```text
chapter-material-package/
├── README.md
├── 正文可写层/
│   ├── 写作总纲.md
│   ├── 分节起草卡.md
│   └── 图表建议清单.md
├── 结果分析层/
│   ├── 关键结果分析卡.md
│   ├── 结果边界卡.md
│   └── 本章小结卡.md
└── 后台核对层/
    ├── 参数核对表.md
    ├── 流程核对总表.md
    └── 图产物索引.md
```
